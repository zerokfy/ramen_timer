* 階層構造
このリポジトリでは、下記のような階層構造を作っている。

+ uvm_env
+-- uvm_agent
+---- uvm_driver
+---- uvm_sequencer
+---- uvm_monitor

* UVMメソドロジークラス
代表的なものは下記。
メソドロジークラスはメソッドを必要とする。
メソッドは各phaseで実行制御される。
# uvm_driver
# uvm_sequencer
# uvm_env
# uvm_agent
# uvm_test
# uvm_monitor
# uvm_scoreboard

* 実行順序
コンポーネントを実行するタイミングをphaseと言う。
phaseは下記順序で実行される。
phaseはfunctionで定義されるが、run_phaseのみ時刻を扱うのでtaskで定義される。
# build_phase
# connect_phase
# end_of_elabration_phase
# start_of_simulation_phase
# run_phase
# extract_phase
# check_phase
# report_phase

* static な method には "::" でアクセスする
uvm_config_db#(uvm_object_wrapper)::set(this,"example", "example_seq", value);
